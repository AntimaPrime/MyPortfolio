Project: Business Financial Data Cleaning & Analysis

July 13, 2025 Anthony Rosado

Introduction:

In the world of data, raw numbers rarely tell the full story. This project began with a cluttered and inconsistent financial dataset from Kaggle.com, the kind that mirrors real-world challenges faced by analysts every day.

Goal:

My goal was to transform this unrefined data into a clean, reliable source of information that could be used to create an interactive dashboard that shows financial trends and actionable insights. This project demonstrates my ability to:

  • Work with imperfect, real-world data.
  • Apply structured data cleaning techniques.
  • Build meaningful visualizations that drive business insights.

Process:

Data Acquisition:

Data was acquired from Kaggle.com and imported into Microsoft Excel. The dataset contained transactional sales data that included revenue, expenses, profit, region, segment, and product information.

Initial Exploration & Inspection:

Reviewed data for completeness and consistency, such as:

  • Missing values
  • Incosistent data types
  • Extra spaces
  • Case mismatches
  • Date format error
Data Cleaning:
  1. Removed duplicate data and data with null values in key fields.
  2. Standardized text fields by removing whitespaces.
  3. Standardized numerical fields by removing erroneous commas and changing data types.
  4. Validated numerical fields to ensure consistency.
Data Transformation:
  1. Reformatted date fields into US Standard MM/DD/YYYY for consistency.
  2. Profit margin column created.
  3. Exported cleaned data as .xlsx for use in Microsoft Power BI.
Data Visualization Using Power BI:
  • Data was uploaded to Power BI, setting data types and initially validated by cross-referencing original dataset.
  • Key Performance Indicators (KPIs) were determined and visualizations were constructed using drill-down effects
  • The dashboard is interactive and adjusts visualizations based on selected criteria.
  • Dashboard higlights:
    • Revenue and profit trends by region and product type.
    • Profitabilty comparisons both yearly and quarterly.
    • Customer segment and sales performance.
Visualization Review:
  • Revenue, COGS, and Profit Trend Over Time
    • Purpose: To visualize monthly or quarterly changes in revenue and profit.
    • Insight: Identified seasonal fluctuations and pinpointed periods of strong or weak financial performance.
    • Evaluation: The line chart format made it easy to compare trends side-by-side and track performance over time.
  • Revenue and Profit by Segment
    • Purpose: To break down revenue, cost, and profit by customer or business segment.
    • Insight: Highlighted which customer segments were most profitable vs. which had high revenue but lower margins.
    • Evaluation: The segmented visuals allowed stakeholders to easily identify the most valuable markets.
  • KPI Cards and Slicers
    • Purpose: To give a snapshot of key performance indicators.
    • Insight: Enabled quick, high-level assessment.
    • Evaluation: KPI cards were effective for executive summary-level viewing, while slicers allowed for user-driven exploration.

Challenges:

  1. Inconsistent Data Formatting - Used Excel functions such as TRIM(), LEFT()/RIGHT(), and Find & Replace.
  2. Missing and Null Values - Removed rows with key information missing.
  3. Duplicated Data - Removed duplicate data using Remove Duplicates tool.
  4. Data Type Mismatches - Converted data types in Excel and Power BI.
  5. Lack of Predefined KPIs - Determined what information would give most actionable insights for data.

Take Aways:

  1. Real-world data often requires cleaning and/or transformation.
  2. Undrestanding business context is important for analysis.
  3. Data analytics is an iterative process.
